```
CONTROL: .word32 0x10000
DATA:    .word32 0x10008

Set CONTROL = 1, Set DATA to Unsigned Integer to be output
Set CONTROL = 2, Set DATA to Signed Integer to be output
Set CONTROL = 3, Set DATA to Floating Point to be output
Set CONTROL = 4, Set DATA to address of string to be output
Set CONTROL = 5, Set DATA+5 to x coordinate, DATA+4 to y coordinate, and DATA to RGB colour to be output
Set CONTROL = 6, Clears the terminal screen
Set CONTROL = 7, Clears the graphics screen
Set CONTROL = 8, read the DATA (either an integer or a floating - point) from the keyboard
Set CONTROL = 9, read one byte from DATA, no character echo.
```

通过设置 `CONTROL` 和 `DATA` 这两个32位内存字来实现不同功能：

1. **内存地址定义**
    - `CONTROL: .word32 0x10000`：定义了一个名为 `CONTROL` 的32位内存字，其地址为 `0x10000`。
    - `DATA: .word32 0x10008`：定义了一个名为 `DATA` 的32位内存字，其地址为 `0x10008`。

2. **功能说明**
    - **输出无符号整数**：当将 `CONTROL` 设置为 `1` 时，系统会将 `DATA` 所指向的值作为无符号整数进行输出。
    - **输出有符号整数**：把 `CONTROL` 设为 `2`，系统会把 `DATA` 对应的值当作有符号整数输出。
    - **输出浮点数**：若 `CONTROL` 等于 `3`，系统将 `DATA` 处的值视为浮点数并输出。
    - **输出字符串地址**：设置 `CONTROL` 为 `4`，`DATA` 应存储要输出字符串的地址。
    - **图形相关输出**：当 `CONTROL` 为 `5` 时，`DATA + 5` 存储 `x` 坐标，`DATA + 4` 存储 `y` 坐标，`DATA` 存储要输出的RGB颜色值。
    - **清空终端屏幕**：`CONTROL` 设为 `6`，系统会清空终端屏幕。
    - **清空图形屏幕**：`CONTROL` 为 `7`，系统将清空图形屏幕。
    - **从键盘读取数据**：`CONTROL` 设置为 `8`，系统会从键盘读取一个整数或浮点数到 `DATA` 指向的位置。
    - **读取一个字节**：`CONTROL` 为 `9`，系统从 `DATA` 指向的位置读取一个字节，并且不会有字符回显。
